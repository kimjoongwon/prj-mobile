## 개발 환경 설정

Plate 개발 환경을 설정하려면 시스템이 필수 사양을 충족하는지 확인해야 합니다. 이 프로젝트는 Expo 및 React Native를 사용하여 구축되어 iOS, Android, 웹 플랫폼 전반에 걸친 크로스 플랫폼 개발을 가능하게 합니다.

### Node.js 및 패키지 관리자 요구사항

프로젝트가 제대로 실행되려면 **Node.js 16+**가 필요합니다. 다음 명령으로 Node.js 버전을 확인할 수 있습니다:

```bash
node --version
```

Node.js를 설치하거나 업그레이드해야 하는 경우 [공식 Node.js 웹사이트](https://nodejs.org/)를 방문하세요.

패키지 관리자로 **npm** 또는 **yarn**을 사용할 수 있습니다. 프로젝트는 `package.json` 파일에 최소 버전을 지정합니다:

- Node.js: >=22.0.0
- npm: >=10.0.0

다음 명령으로 프로젝트 의존성을 설치하세요:

```bash
npm install
```

이 명령은 Expo, React Native, 내비게이션 라이브러리, 상태 관리 도구 및 UI 컴포넌트를 포함하여 `package.json`에 나열된 모든 의존성을 설치합니다.

## 플랫폼별 요구사항

### iOS 개발 요구사항

iOS용 개발을 위해서는 **Xcode**가 설치된 **Mac**을 사용해야 합니다. Xcode는 iOS 애플리케이션을 빌드하고 테스트하는 데 필요한 도구와 시뮬레이터를 제공합니다.

Mac App Store에서 Xcode를 설치하세요. 설치 후 Xcode를 열어 라이선스 계약을 수락하고 요청되는 경우 추가 구성 요소를 설치하세요.

프로젝트의 `app.json`은 다음과 같은 iOS 특정 설정을 구성합니다:

- 번들 식별자: `com.anonymous.plate`
- 태블릿 기기 지원
- 앱 아이콘 및 스플래시 화면 구성

### Android 개발 요구사항

Android 개발의 경우 **Android Studio** 또는 독립형 **Android SDK**가 필요합니다. Android Studio는 Android 앱 빌드에 필요한 Android 에뮬레이터 및 개발 도구를 제공합니다.

[공식 웹사이트](https://developer.android.com/studio)에서 Android Studio를 설치한 다음 AVD Manager를 통해 테스트용 Android Virtual Device(AVD)를 설정하세요.

`app.json` 파일에는 다음과 같은 Android 특정 구성이 포함되어 있습니다:

- 패키지 이름: `com.anonymous.plate`
- 전경, 배경 및 모노크롬 이미지가 있는 적응형 아이콘
- 엣지 투 엣지 디스플레이 활성화
- 예측 뒤로 제스처 비활성화

## 개발 설정 워크플로우

필수 도구 및 의존성을 설치한 후 Expo의 CLI를 사용하여 개발 서버를 시작할 수 있습니다.

### 개발 서버 시작

다음 명령으로 개발 서버를 시작하세요:

```bash
npm start
```

이 명령은 Metro bundler를 시작하며, 애플리케이션 코드를 제공하고 빠른 개발 피드백을 위한 Hot Module Replacement(HMR)를 지원합니다. 번들러는 컴파일된 모듈을 캐시하여 시작 성능을 향상시킵니다.

`npm start`를 실행하면 QR 코드와 다양한 플랫폼에서 실행할 수 있는 옵션이 표시됩니다.

**`npm start`를 사용하는 시기:**

- 앱을 처음 시작할 때
- HMR이 올바르게 작동할 때
- Metro bundler 캐시 문제가 없을 때

**섹션 출처**

- [README.md](file://README.md#L48-L61)
- [package.json](file://package.json#L34)

### Metro Bundler 캐시 지우기

오래된 코드나 스타일이 적용되지 않는 문제가 발생하면 Metro bundler 캐시를 지워야 할 수 있습니다:

```bash
npm run start:clear
```

이 명령은 `--clear` 플래그와 함께 개발 서버를 시작하여 Metro가 캐시된 버전을 사용하는 대신 모든 모듈을 처음부터 다시 빌드하도록 합니다.

**`npm run start:clear`를 사용하는 시기:**

- Metro bundler 캐시 문제로 `npm start`가 작동하지 않을 때
- 새 의존성을 추가한 후
- 스타일 또는 구성 변경이 적용되지 않을 때
- Metro 오류가 발생했을 때

`npm start`와 `npm run start:clear` 중 선택하기 위한 결정 플로우:

```
npm start → 오류 발생?
   ↓ YES
npm run start:clear → 해결?
   ↓ YES: 계속 사용
   ↓ NO: Prebuild 섹션 참조
```

## 애플리케이션 실행

`package.json`에 정의된 특정 npm 스크립트를 사용하여 다양한 플랫폼에서 애플리케이션을 실행할 수 있습니다.

### iOS에서 실행

iOS 시뮬레이터에서 애플리케이션을 실행하려면:

```bash
npm run ios
```

캐시 지우기와 함께 실행하려면:

```bash
npm run ios:clear
```

이 명령은 내부적으로 Expo의 `expo run:ios`를 사용하여 앱을 빌드하고 iOS 시뮬레이터에서 실행합니다.

이 명령은 Expo의 `expo run:android` 명령을 사용하여 앱을 빌드하고 Android 에뮬레이터에서 실행합니다.

## 일반적인 문제 해결

### Metro Bundler 캐시 문제

개발 중 가장 일반적인 문제 중 하나는 Metro bundler 캐시가 오래된 코드를 제공하는 것입니다. 증상은 다음과 같습니다:

- 파일을 저장한 후 코드 변경사항이 반영되지 않음
- 스타일 업데이트가 표시되지 않음
- 이전 컴포넌트 동작이 지속됨

**해결책:**

다음을 사용하여 Metro bundler 캐시를 지우세요:

```bash
npm run start:clear
```

이렇게 하면 Metro가 모든 모듈을 다시 컴파일하도록 강제하고 캐시 관련 문제를 제거합니다.

**섹션 출처**

- [README.md](file://README.md#L619-L630)

### 네이티브 프로젝트 생성 오류

"Failed to create native project" 오류가 발생하면 일반적으로 캐시 문제 또는 구성 충돌 때문입니다.

**해결책:**

clean prebuild 명령을 사용하세요:

```bash
npm run prebuild:clean
```

이렇게 하면 `ios/` 및 `android/` 디렉토리를 완전히 제거하고 `app.json`의 Expo 구성을 기반으로 처음부터 다시 생성합니다.

**섹션 출처**

- [README.md](file://README.md#L607-L617)

### 의존성 설치 문제

특히 네이티브 모듈이 있는 새 의존성을 설치한 후 앱이 시작되지 않을 수 있습니다.

**해결책:**

clean prebuild를 실행하세요:

```bash
npm run prebuild:clean
```

많은 의존성에는 네이티브 코드 통합이 필요하며, 새 prebuild는 적절한 구성을 보장합니다.

### 플랫폼별 문제

특정 플랫폼에서 문제가 발생하는 경우:

**iOS 문제의 경우:**

```bash
npm run prebuild:ios:clean
```

**Android 문제의 경우:**

```bash
npm run prebuild:android:clean
```

이 명령은 플랫폼별 네이티브 프로젝트만 재생성하여 플랫폼별 구성 문제를 해결할 수 있습니다.
